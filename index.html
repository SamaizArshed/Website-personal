I will write a paragraph of each methodology of SDLC( SDLC ). There are many of them but I will suggest a few.My suggestions and reasons are:


Waterfall Method

The waterfall method as the name suggests happens in phases that trickle down. This proess was developed for industrial projects but is also used in software development. You only move on to the next phase when the previous step is fully completed.Important phases or waterfall steps include requirements, design , development, testinf and finally deployment. These steps change accpording to industry but concept reamions the same. Adcantages include having clarity and focus on each phase with clearly defined goals that allow the process to flow swquentially. It is helpful for small projects and less costly. Another version of this is called the V SHAPED MODEL where each phgase of waterfall model is also tested and validated side by side thus creating a v shape of processess. This is an extension of waterfall as the processes remain similar with added testing.


Spiral Methodology

This model is an evolution of the waterfall model and focuses on iteration. this means that we have 4main processes tht are repeated again and again as per requirements which results in a circular or spiral shape. the 4 phases are objectives defined, risk analysis and prototyping, design and engineering and review or evaluation phase. We can move in between the 4 phases in steps and then go again in a circle to test if our product manager requires.It is much more flexible and suitable for large projects and main thing here is the analysis part which reduces risk in the software development lifecycle.


Agile Methodology

Argubly the most popular model used in software-development-life-cycle-sdlc today is known as Agile methodology and as the name suggests, it relies on speed. In todays market, customers cant wait for months or even year for a software project to get delivered comlpetely as new products are released and sometimes the changes happen after one phase of project. The key here is speed and responsivness.So the different phases of the agile methodology including design,planning, develoment, testing or evaluation happens continuausly that incorporates feedback from the customers and peers. This results in increased collaboration and communication among stakeholders and results in flexibility for team members. Any changes can be added during any phase and this makes the model adaptable.This allows the product to be deployed in the market at an increased speed and allows chan as per new requirements.


Among so many sdlc-methodologies, the ones that are in the Agile classification are better. Among Agile methods, I also have few suggestions:


Scrum

Scrum is one of the types of Agile methodology.It is a lightweight and incremental approach which allows the team members to work efficiently together. A concept here is SPRINT which means a set time for doing tasks. So tasks are divided into sprints usually for 2 weeks that allow tasks to be completed within the time and then delivering an reviewing before the next sprint starts.The main advantage here is that is resource efficient and fast moving.There are also regular reviews known as sprint retrospectives or stand ups daily to check on progess and ensure efficiency.Main disadvantage here of scrum is that it does not allow much changes within a sprint. So if design for a sprint is not done properly, 2 weeks can be lost. Yet the approach also allows for flexibility after each sprint. This method is widely used in startups and tech companies to increase productivity.


Kanban Method

Another approach in Agile is knowna s Kanban. Here we can start anywhere with an exisiting project and unlike scrum there is no specific role defined. Rather from the beginning the tasks are divided into sections on the Kanban board such as backlog, in progress, to be delivered today etc. This allows for continuous and incremental changes resulting in fast decisions.There is accountability and leardership at all levels as tasks are divided among team members who must take responsibility for their completion. The kanban board also allows all team members to view current progress of peers and review them periodically as per requirements.


Crystal Agile Method

This method focuses more on individuals rather than on tasks and projects.As new projects are always eveolving and changing, the focus is on teams whcih can deliver the best results.This adaptive approach lets the team to respond to changes frequently. The teams are divided into parts such as white, yellow, orange and each section has its own properties e.g no. of team members or no. of months needed and depending on the client requirements, we can select the best crystal team on that project.The framework promotes collaboration and increases productivity.

I have used CSS(Cascading Style Sheets) in this project. There are 3 ways to add CSS to html files:


1.External

External method of CSS is used when we want to keep a separate file of styling seaprate from html file . Here a new .css file is added and all the styling is done externally in the .css file. this ensures that all styling is in a different place that can be changed without changing the html.


2.Internal

As the name suggests, we use CSS in the html file but put it is separate from the html code. Style tags are used inside head section and so all styling is in one place within the same file. It is different from external as there is no separate .css file created but rather the same file is used for styling.


3.Inline

In this method, the html code is written side by side with the css styling using the style tag. It is quick and very easy to understand for others as each styling is done where the html code is and makes it easier to edit indiviual elements.However as each style is linked directly, it is not reusable for other parts directly.

The reason CSS is used in a webpage is:


Cascading style sheets as the name suggests is used for styling webpages and making them responsive on different devices. User experience is enhanced when elements on the webpage are nicely edited and easy to read. All such styling is done by CSS which can be reused when using external css and it increases consistency of design throughout the website.Its also easier for others to check colours, fonts etc when reviewing css files.
